# Dumate Studio v5 · 启动与架构说明

> 2026-06-12 · v5 全量重构版（旧版完整备份在 `_backup_v4/`，数据库独立、随时可回退）

## 启动

```bash
# 方式一：双击 start.command（推荐）
# 方式二：
node tools/serve.mjs 4173        # 或 python3 -m http.server 4173
# 浏览器打开 http://localhost:4173
```

⚠️ v5 使用 ES Modules，**不能双击 index.html 用 file:// 打开**，必须走本地 http 服务。

## v5 相对 v4 的变化

| 维度 | v4 | v5 |
|------|----|----|
| 代码组织 | 单文件 app.js 4257 行 | 34 个 ES Module（core / domain / api / agent / views / ui） |
| 任务模型 | 手动任务 + 快速任务两套互不相通 | 统一 `production` + 阶段状态机（脚本→分镜/成图→提示词→生成→剪辑→文案→审核→交付） |
| Agent | 日志气泡 + setInterval 盯进度，刷新即断 | 独立沉浸式工作台：会话持久化、结构化卡片（计划/进度/回传/审核/结果/失败）、事件驱动编排器、**刷新后批次自动接续** |
| 视频生成 | 同步假调用 | Job 队列 + ProviderAdapter（排队/进度/失败/重试/取消全状态），Mock 引擎模拟真实异步；接真 API 只需新增一个 adapter |
| 剪辑 | 基础时间轴 | 智能拼接（自动按片段排序 + 从口播铺字幕）+ 时间轴精修（排序/裁剪/**分割**/字幕拖拽/样式/SRT/撤销/缩放） |
| 存储 | IndexedDB 整包快照（生成中任务刷新即丢） | 分仓存储（8 个 store）+ 图片 Blob 化 + 自动迁移 v4 数据 + 导出/导入 |
| 措辞 | 发布 | **交付 / 定稿入库**（内部归档 + 供应商分发，无任何平台发布行为） |
| UI | 三套视觉混用 + emoji 图标 | 统一 tokens、SVG 图标、图标导航外壳、⌘K 命令面板、通知中心 |

## 信息架构

```
登录分流（创作端 / 供应商）
├── 总览        待办导向：待回传 / 渲染中 / 待审核 / 失败 / 待下载
├── Agent 工作台 沉浸式三栏：会话 | 对话流（结构化卡片）| 任务看板（流水线圆点）
├── 创作空间    账号面板 + 链路工具页（统一 stepper）
│   ├── 脚本   AI 按定位生成 / 行内编辑 / 优化方向重写
│   ├── 分镜·成图  统一参考图 + 站外整段提示词 + 拖拽回传（图片 API 未接入期的真实出图通道）
│   ├── 提示词  两段独立 15s 框架（数字人 / 无数字人）
│   ├── 生成台  即梦式对话流 · Job 驱动 · @资产 · 模拟引擎
│   ├── 剪辑   智能拼接 + CapCut 式时间轴
│   ├── 文案   爆款标题 + 正文
│   └── 审核   通过 / 驳回退回 / 交付入库
├── 资产库      全账号聚合 · 标签 · 拖拽上传
├── 交付中心    创作端视角（全链路回看）/ 供应商视角（表格 + 批量 zip）
└── 设置        能力-Key 档案 · 测试连接 · 数据导出/导入/清空
```

## 接真实视频 / 图片 API（预留好的口）

1. 在 `js/api/providers.js` 仿照文件底部的模板新增 adapter（实现 `submit / poll / cancel` 三个方法 + 字段映射）；
2. `registerProvider(...)` 注册，`activeProviderFor()` 会在设置页配置了对应 Key 后自动路由；
3. UI / 队列 / 重试 / Agent 编排零改动。

语言模型：内置 DeepSeek 直连（内部原型约定）；设置页保存"脚本/文案"类 Key 可覆盖 endpoint / key / model；CORS 受阻时 `python3 proxy.py` 后把 Provider 填 `http://localhost:8787/chat`。

## 数据与回退

- v5 数据库：`dumateStudioV5`（IndexedDB 分仓）；首次启动自动从旧库 `acgVideoTool` 迁移（账号 / 资产 / 历史交付 / 在制任务 / 快速批次），**旧库原样保留**。
- 回退方式：把 `_backup_v4/` 三件套复制回根目录即可（旧库数据未动过）。
- 设置页支持全量 JSON 导出 / 导入 / 清空。
