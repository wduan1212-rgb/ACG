# Dumate Content Factory 本地预览入口（v3）

## 启动方式

在当前文件夹执行：

```bash
python3 -m http.server 4173
```

然后浏览器打开 `http://localhost:4173`，或直接双击同目录下的 `index.html`。

## v3 这一版做了什么

整套界面与交互重做。视觉上「浅色为主 + 视频生成台深色」，全流程 6 个页面打通：
**账号首页 → 创建账号（含图片资产）→ 创作空间 → 脚本生成 → 提示词生成 → 即梦式视频生成台 → 粗剪导出 / 资产 / 设置。**

核心链路严格按你描述的流程实现：

1. **账号分组 + 定位**：左栏按图文组 / 视频组 / 混合组分类；每个账号带定位、平台、模式、语气。
2. **创建账号即绑定图片资产**：创建对话框里有「账号图片资产」区，上传的图后续在生成台可直接 @ 调用。
3. **脚本生成**：AI 依「账号定位」生成口播稿，并自动按 15s 拆段（右栏显示分段结构）。
4. **提示词生成**（独立一步）：每段拆成视频提示词；超过 15s 自动拆两段，套用数字人结构——
   **前 15s 数字人出镜 + 产品演示，后 15s 产品演示延续 + 数字人出镜**；含 UI 镜头的段会提示绑定截图参考。
5. **即梦式视频生成台**（深色）：左侧片段/历史，中间生成结果对话流，底部悬浮输入框；
   支持上传图片做全能参考、拖拽 / 粘贴图片、@ 资产库、UI 锁定、比例/时长切换，生成版本可「加入粗剪」。
6. **粗剪导出**：时间线拖拽排序，一键生成规范导出命名。

脚本内容围绕「百度搭子 Dumate」功能特点做延展教学。

## 新增（v3.1）

**1. 即梦式内联 @ 资产**
视频生成台的输入框已改成富文本：@ 资产或上传/拖拽/粘贴图片时，会以「小缩略图 + 名字」的芯片形式直接出现在文字中间（点 × 可删），和即梦工作台一致。发送给模型时芯片自动序列化成 `@资产名`。

**2. 已接入语言模型（DeepSeek）生成提示词**
脚本生成、提示词生成已接 DeepSeek（OpenAI 兼容协议），默认 key 已内置。
提示词严格按你给的「前15秒 / 后15秒」两段式框架输出（数字人开场 + 产品界面引入 + 任务拆解 / 执行结果 + 数字人收束 + 负面提示词），每 30 秒一个场景。

- 模型配置在 `app.js` 顶部 `CONFIG`（endpoint / model / apiKey），也可在浏览器控制台改：`DumateConfig.model = "deepseek-reasoner"`。
- 在「设置 · API 配置」里把「服务类型」选**脚本生成 API**保存新 Key，会实时覆盖 `CONFIG.apiKey`；Provider 填成 `http://localhost:8787/chat` 可切到本地代理。
- 调用失败（网络 / CORS / Key 错误）会自动回退本地模板，并在提示里说明原因，页面不会卡住。

**如果浏览器报 CORS 错误**（直连被拦），在本文件夹另开一个终端运行：

```bash
python3 proxy.py
```

然后在浏览器控制台执行 `DumateConfig.endpoint = "http://localhost:8787/chat"`，再重新生成即可。代理只是把请求转发到 DeepSeek 并补上跨域头。

> 注意：当前 key 写在前端代码里，仅适合内部原型；对外发布前请改为后端调用，避免 key 暴露。

## 新增（v3.2）

- **脚本改成结构化分镜表**：脚本不再是大段文字，而是一张可编辑表格。视频模式列为「时间 / 核心思想 / 画面分镜 / 口播」；图文模式自动去掉口播，列为「核心思想 / 画面分镜 / 图上文案」。单元格点开即改，可加/删镜头。

- **下一步据脚本生成**：视频提示词、图片提示词都改成读取这张分镜表的内容来生成。
- **修复**：视频生成台切换「前15s / 后15s」时，输入框现在会正确载入各自的提示词（之前两段显示同一段）；切回会保留你的编辑。

## 真实 API 接入点

所有模型调用收敛在 `app.js` 末尾的 `window.DumateAPI`：

```text
DumateAPI.generateScript()   // 脚本生成（语言模型）
DumateAPI.generatePrompts()  // 提示词拆分
DumateAPI.generateVideo()    // 视频生成（即梦 / Seedance / 千帆）
DumateAPI.generateImage()    // 图片 / 角色版生成
```

当前返回示例数据。接入时在「设置 · API 配置」保存对应 Key，再把每个方法体替换成真实 `fetch`，**保持返回结构不变即可，UI 层无需改动**。
